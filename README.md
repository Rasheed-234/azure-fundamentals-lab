# 2. Azure Free Account Creation

## Overview

The first stage of this project involved creating a Microsoft Azure Free Account. This account provides access to Microsoft Azure services, enabling users to explore cloud computing through hands-on practical experience. Microsoft offers new users free credits and access to selected services at no cost for a specified period, making it ideal for learning and experimentation.

Creating the Azure account required completing Microsoft's identity verification process, which included email verification, phone verification, and payment card verification. These verification steps help Microsoft maintain the security and integrity of the Azure platform while preventing fraudulent account creation.

---

## Objective

The objective of this task was to:

- Gain access to the Azure Portal for cloud resource management.

---

## Step-by-Step Account Creation Process

### Step 1: Visit the Azure Website

Open a web browser such as Microsoft Edge or Google Chrome and navigate to the Microsoft Azure website.

**Azure Portal:**

```
https://portal.azure.com
```

Alternatively, visit:

```
https://azure.microsoft.com/free
```

Click the **Start Free** button to begin the registration process.

---

### Step 2: Sign In or Create a Microsoft Account

If you already have a Microsoft account, sign in using your existing credentials.

If you do not have one, select **Create one** and provide:

- First Name
- Last Name
- Email Address
- Password

After entering the required information, Microsoft creates your account and proceeds to the verification stage.

---

### Step 3: Email Verification

Microsoft sends a verification code to the email address used during registration.

To complete this process:

1. Open the registered email inbox.
2. Locate the verification email from Microsoft.
3. Copy the verification code.
4. Return to the Azure registration page.
5. Enter the code in the verification field.
6. Click **Verify**.

Successful email verification confirms ownership of the email address and allows the registration process to continue.

---

### Step 4: Phone Number Verification

The next stage requires verification of an active mobile phone number.

Steps performed include:

1. Select the country or region.
2. Enter a valid mobile phone number.
3. Choose the preferred verification method (SMS or voice call).
4. Receive the One-Time Password (OTP).
5. Enter the OTP into the Azure verification page.
6. Click **Verify Code**.

Phone verification helps Microsoft prevent automated registrations and ensures that each Azure account is associated with a valid user.

---

### Step 5: Identity Verification Using a Payment Card

To complete identity verification, Microsoft requests a valid debit or credit card.

The following information is entered:

- Cardholder Name
- Card Number
- Expiry Date
- Security Code (CVV)

Microsoft performs a temporary authorization check to confirm that the card is valid.

> **Note:** The card is used only for identity verification during the registration process. Charges are only incurred if the user intentionally upgrades to paid services or exceeds the Free Tier limits.

---

### Step 6: Accept Terms and Conditions

Before creating the Azure subscription, Microsoft requires users to review and accept the following:

- Microsoft Azure Terms of Service
- Microsoft Online Services Agreement
- Privacy Statement

After reading these documents, click **Accept** to continue.

---

### Step 7: Create the Azure Subscription

Once all verification steps are completed successfully, Microsoft automatically provisions an Azure Free Subscription.

This subscription grants access to:

- Azure Portal
- Azure Resource Manager
- Resource Groups
- Storage Accounts
- Virtual Machines
- Azure Networking
- Cost Management
- Azure Monitor
- Other eligible Azure services

---

## Azure Free Tier Benefits

After successful registration, the Azure Free Account provides access to:

- Free Azure credits for eligible services.
- Always Free Azure services with monthly usage limits.
- Ability to deploy and manage cloud resources.
- Access to Azure learning resources and documentation.
- Monitoring tools for usage and spending.

These benefits provide an excellent environment for learning Microsoft Azure without incurring unnecessary costs when resources are managed responsibly.

# Budget Management

Azure Budgets allow users to define spending limits for subscriptions or resource groups.

A budget does not stop Azure resources from running; instead, it monitors spending and generates notifications when predefined thresholds are reached.

Budgets are useful because they help users:

- Prevent overspending.
- Monitor Azure credits.
- Receive early warning notifications.
- Improve cost planning.
- Manage cloud resources responsibly.

# Why Use a 75% Budget Alert?

A 75% budget alert provides an early warning before the full budget is exhausted.

Benefits include:

- Prevents unexpected charges.
- Encourages responsible resource usage.
- Gives enough time to delete unused resources.
- Helps students remain within Azure Free Tier limits.
- Supports better financial planning.

Using alerts is considered a cloud best practice because it allows proactive cost management rather than reacting after overspending has occurred.

---

# Best Practices for Azure Cost Management

To manage Azure costs effectively, the following best practices should be followed:

- Review Cost Analysis regularly.
- Configure budgets for every subscription.
- Set alert thresholds (50%, 75%, and 90% are common choices).
- Delete unused resources.
- Stop virtual machines when they are not in use.
- Monitor Azure Advisor recommendations.
- Use Azure Free Tier services whenever possible.

# Multi-Factor Authentication (MFA)

## What is MFA?

Multi-Factor Authentication (MFA) is an additional layer of security that requires users to verify their identity using two or more authentication methods.

Instead of relying only on a password, Azure requires another verification factor before granting access.

Common authentication methods include:

- Microsoft Authenticator App
- SMS verification code
- Phone call verification
- Security key (FIDO2)
- Biometric authentication (where supported)

MFA significantly reduces the risk of unauthorized account access, even if a password has been compromised.

# Why MFA is Important

Enabling MFA provides several benefits:

- Protects Azure resources from unauthorized access.
- Prevents attacks using stolen passwords.
- Enhances account security.
- Meets Microsoft's recommended security practices.
- Reduces the risk of phishing and credential theft.

Microsoft recommends enabling MFA for all Azure accounts, especially those with administrative privileges.

- At least 12 characters long.
- Unique for each online account.
- Difficult to guess.
- Not based on personal information.

A strong password should contain a combination of:

- Uppercase letters
- Lowercase letters
- Numbers
- Special characters

| Issue | Possible Cause | Solution |
|------|----------------|----------|
| Verification email not received | Email delay or spam filtering | Check Spam/Junk folder and request a new verification code. |
